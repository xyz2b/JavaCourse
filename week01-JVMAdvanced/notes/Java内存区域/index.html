Java内存区域

概览

程序计数器(Program Counter Register)

作用:

  • 当前线程所执行的字节码的行号指示器
  • 字节码解释器工作时就是通过改变这个计数器的值来选取下一条需要执行的字节码指令

内容:

  • 如果线程正在执行的是一个Java方法，这个计数器记录的是正在执行的虚拟机字节码指令的地址
  • 如果执行的是本地(Native)方法，这个计数器值则应为空(Undefined)

溢出:

  • 此内存区域是唯一一个在《Java虚拟机规范》中没有规定任何OutOfMemoryError情况的区域

属性:

  • 线程私有内存

虚拟机栈(VM Stack)

作用:

  • 虚拟机栈描述的是Java方法执行的线程内存模型
  • 每个方法被执行时，Java虚拟机都会同步创建一个栈帧(Stack Frame)用于存储局部变量、操作数栈、动态链接、方法出口等信息
  • 每一个方法被调用直至执行完毕的过程，就对应着一个栈帧在虚拟机栈中从入栈到出栈的过程

内容:

  • 局部变量表(Local Variable Table)
  • 操作数栈
  • 动态链接
  • 方法出口

溢出:

  • 如果线程请求的栈深度大于虚拟机所允许的深度，将抛出StackOverflowError异常
  • 如果Java虚拟机栈容量可以动态扩展，当栈扩展时无法申请到足够的内存会抛出OutOfMemoryError异常

属性:

  • 线程私有内存

局部变量表(Local Variable Table)

作用:

  • 保存局部变量
  • 保存计算中间结果

内容:

  • Java基本数据类型: boolean、byte、char、short、int、float、long、double
  • 对象引用: reference类型，它并不等同于对象本身，可能是指向对象其实地址的指针，也可能是指向一个代表对象的句柄或者其他与此对象相关的位置
  • returnAddress类型: 指向一条字节码指令的地址

属性:

  • 局部变量表中的存储空间以局部变量槽(Slot)表示，其中64位长度的long和double类型的数据会占用两个槽，其余数据类型只占用一个
  • 编译期确定大小

操作:

  • 给局部变量赋值时，需要使用store指令进行操作，如astore_1。store类指令都会删除栈顶值
  • 相应load指令则会将值从局部变量表压入操作数栈，但并不会删除局部变量表中的值

操作数栈

作用:

  • 有一些操作码可以将值压入栈
  • 还有一些操作码则是从栈中获取操作数，并进行处理，再将结果压入栈
  • 接收调用其他方法返回的结果值

内容:

  • 基本所有的数值运算和类型转换指令都需要将数值load到操作数栈才能进行操作
  • 调用其他方法的返回值

属性:

  • 唯一不需要将数值load到操作数栈的指令是iinc，它可以直接对LocalVariableTable中的值进行运算。其他所有操作都是使用操作数栈来执行的
  • 编译期确定大小

操作:

  • load

本地方法栈(Native Method Stack)

作用:

  • 与虚拟机栈所发挥的作用相似，区别只是虚拟机栈为虚拟机执行Java方法(也就是字节码)服务，而本地方法栈则是为了虚拟机使用到的本地(Native)方法服务

属性:

  • HotSpot虚拟机直接将本地方法栈和虚拟机栈合二为一

溢出:

  • 与虚拟机栈一样，本地方法栈也会在栈深度溢出或者栈扩展失败时分别抛出StackOverflowError和OutOfMemoryError异常

堆(Heap)

以JDK8的GC方式为例

作用:

  • 存放对象实例(几乎所有的对象实例以及数组都应当在堆上分配)

内容:

  • 新生代
    • Eden区
      • TLAB(Thread Local Allocation Buffer): 线程私有的分配缓冲区
    • From Survivor
    • To Survivor
  • 老年代
  • 字符串常量池

溢出:

  • 如果在Java堆中没有内存完成实例分配，并且堆也无法再扩展时，Java虚拟机将会抛出OutOfMemoryError异常

属性:

  • 依据使用的GC不同，堆内存的区域划分而有所差别
  • 当前主流的Java虚拟机都是按照可扩展来实现堆的(通过参数-Xmx和-Xms设定)
  • 线程共享内存

方法区(Method Area)

作用:

  • 用于存储已被虚拟机加载的类型信息、常量、静态变量、即时编译器编译后的代码缓存等数据

内容:

  • 运行时常量池: 方法引用、域引用、类引用、字符串值、字面量
  • 字段信息: 字段名、字段类型、字段修饰符
  • 方法信息: 修饰符、方法名、参数名、方法参数类型、方法返回值、异常表、字节码
  • 类变量(静态变量)
  • 方法表
  • 类型信息: 类全限定名、父类全限定名、修饰符
  • 类中的特殊方法: 实例初始化方法<init>、类初始化方法<clinit>
  • 指向Class实例的引用
  • 指向类加载器的引用
  • JIT代码缓存

溢出:

  • 如果方法区无法满足新的内存分配需求时，将抛出OutOfMemoryAError异常

属性:

  • 别名: 非堆(Non-Heap)
  • JDK7: 永久代(属于运行时数据区)；JDK8: 元空间(属于直接内存，不属于运行时数据区)
  • 线程共享内存

运行时常量池(Runtime Constant Pool)

作用:

  • Class文件中除了有类的版本、字段、方法、接口等描述信息外，还有一项信息是常量池，用于存放编译期生成的各种字面量与符号引用，这部分内容在类加载后存放到方法区的运行时常量池中

内容:

  • 方法引用
  • 域引用
  • 类引用
  • 字符串值
  • 字面量

溢出:

  • 由于是方法区的一部分，当然受到方法区的限制，如果常量池无法再申请到内存时，将抛出OutOfMemoryError异常

属性:

  • 方法区的一部分
  • 除了保存Class文件中描述的符号引用外，还会把由符号引用翻译出来的直接引用也存储到运行时常量池中
  • 并非预置入Class文件中常量的内容才能进入方法区运行时常量池，运行期间也可以将新的常量放入池中，这种特性用的比较多的是String类的intern()方法

直接内存(Direct Memory)

作用:

  • 类如在JDK1.4新加入NIO(New Input/Output)类，引入了一种基于通道(Channel)与缓冲区(Buffer)的I/O方式，它可以使用Native函数库直接分配堆外内存，然后通过一个存储在Java堆里面的DirectByteBuffer对象作为这块内存的引用进行操作。这样能够在一些场景中显著提高性能，因为避免了在Java堆和Native堆中来回复制数据

内容:

  • JDK8之后，方法区的实现挪到了直接内存的元空间中

溢出:

  • 无法像操作系统再申请新的内存时，将抛出OutOfMemoryError异常

属性:

  • 不是虚拟机运行时数据区的一部分
  • 直接内存的分配不会受到Java堆大小的限制
  • 会受到物理内存的限制
